前言

Hexo 是一个快速、简洁且高效的博客框架，它使用 Markdown 来编写文章，同时支持多种主题和插件。并且结合 Github Pages 可以完全免费（当然，除了一个属于自己的域名）搭建一个属于自己的个人博客网站。

本文就详细介绍下如何使用 Hexo + GitHub 搭建免费个人博客网站的教程。

0.简介

什么是 GitHub Pages？

  • What is GitHub Pages? - GitHub Help
    GitHub Pages 是由 GitHub 官方提供的一种免费的静态站点托管服务，让我们可以在 GitHub 仓库里托管和发布自己的静态网站页面。

什么是 Hexo？

  • Hexo - 快速、简洁且高效的博客框架
    Hexo 是一个快速、简洁且高效的博客框架，它使用 Markdown 来编写文章，同时支持多种主题和插件。

Hexo + GitHub 文章发布原理

整体流程概述

使用 Hexo 结合 GitHub 发布文章，主要分为本地创作和线上发布两个大阶段。整个过程利用 Hexo 强大的静态网站生成能力和 GitHub Pages 的免费托管服务，让用户可以轻松搭建和发布个人博客。

本地创作阶段

  1. 文章编写：用户使用 Markdown 语法在本地编写博客文章，Hexo 支持直接将 Markdown 文件作为文章源。
  2. 静态页面生成：当文章编写完成后，在命令行中执行 hexo generate 命令，Hexo 会读取 Markdown 文件，依据配置的主题模板，将其转换为静态的 HTML、CSS 和 JavaScript 文件，这些文件会存放在本地的 public 目录下。

线上发布阶段

  1. 文件推送：执行 hexo deploy 命令，Hexo 会借助 Git 将 public 目录下生成的静态文件推送到预先配置好的 GitHub 仓库中。
  2. 网站展示：GitHub Pages 会自动检测仓库中的静态文件，当检测到符合条件的文件后，就会将这些文件渲染成可访问的网页。用户可以通过对应的 GitHub Pages 域名（如 username.github.io）访问自己的博客网站。

6. 配置自定义域名

博客搭建完成使用的是 GitHub 的子域名（用户名.github.io），我们可以为 Hexo 博客绑定自己的域名替换 GitHub 域名，更加个性化和专业，也利于 SEO。

这里使用的是 namesilo 域名注册商(主要便宜且可用支付宝支付)，其他域名注册商配置类似。

6.3 开启 GitHub Pages HTTPS

打开 GitHub Pages 仓库的 Settings 页面，输入你自己的自定义域名，找到 “Enforce HTTPS” 选项，将其勾选上。

HTTPS 证书部署成功需要一定时间，等大概几分钟再访问域名，就可以看到域名下方由DNS Check in Progress 变成了 DNS Check in Success。表明HTTPS 配置完成！

Tips: 若你采用的是Cloudflare DNS 服务，在GitHub Pages 仓库的 Settings 页面中 Enforce HTTPS 选项大概率是灰色不可选的；需要在 DNS 配置中添加 HTTPS 证书后，需要将代理状态设置为 “仅代理” 后才可选择。

结语

Hexo 作为纯静态博客，需要在本地完成文章编辑后再部署到 GitHub，依赖本地环境，不像 WordPress 或 Typecho 这类动态博客能直接在浏览器里撰文和发布，这是一种颇具极客风格的写博客方式。但其免费、稳定又省心的优势也十分明显，很适合爱折腾研究，或是没有在线发文需求的朋友。希望这篇教程能帮你顺利开启这段独特的博客搭建之旅。若在过程中遇到任何问题，或是你有相关建议，欢迎在github留言交流。
